La URI (Identificador de recursos uniforme)

La URI es una cadena de caracteres para Identificar a un recurso en una red.

Datos Importantes de la URI

  • La URI puede ser una URL, URN o ambos
  • La sintaxis de la URI está descrita en RFC 3986
  • Se utiliza para establecer hiperenlaces en la WWW
  • La URI es un sistema diseñado para nombrar y localizar los recursos en la Web.

Sintaxis de la URI

Existen 2 clases de URI:

  • URL (Uniform Resource Locator): Proporciona la localización del recurso.
  • URN (Uniform Resource Name): Proporciona el nombre del recurso.

El protocolo HTTP (HyperText Transfer Control)

Como se vió, la arquitectura web se compone de los elementos: Cliente, Servidor y Protocolo HTTP.

En la web se establece un diálogo entre el Cliente - Servidor con el hin de consumir recursos (hipertexto).

El cliente solicita recursos y el Servidor responde con los recursos solicitados. Así, el cliente y el servidor deben hablar el mismo idioma -> Protocolo HTTP que consta de un conjunto de normas, reglas y formatos.

Características claves de HTTP

  • HTTP es un protocolo sin estado -> No existe el concepto de sesión, el servidor se olvida completamente del cliente al finalizar la conexión y lo tratará como un cliente nuevo
  • HTTP es un protocolo sin estado (stateless).
  • HTTP es un protocolo orientado a la conexión.
  • HTTP es un protocolo de transferencia de hipertexto.
Cliente -> Servidor SOLICITUD HTTP - HTTP REQUEST
Servidor -> Cliente RESPUESTA HTTP - HTTP RESPONSE
